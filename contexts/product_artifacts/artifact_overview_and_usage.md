# Product Artifacts: Overview and Usage

Product Artifacts are the tangible outputs of the product management process. They are living documents that align cross-functional teams, record decisions, and guide the product from mere idea to shipped reality. 

In this workshop, you will progress sequentially through 8 core artifacts. 

## The Core Product Artifacts

### 1. Design Thinking Document
*   **Function:** The genesis of the product. It documents the initial exploration of user empathy, problem definition, and initial ideation. 
*   **How to use:** Use this early in the discovery phase. When the team gets lost in technical details later, bring them back to the "Define" section to remember the human problem being solved.

### 2. User Persona
*   **Function:** A data-driven avatar representing the target user segment.
*   **How to use:** Keep this visible. During design reviews, ask, "Would *[Persona Name]* understand this button?" If not, redesign it. It settles subjective design arguments by focusing on objective user needs.

### 3. User Journey Map
*   **Function:** A visual storyboard of the user's end-to-end experience, spanning from initial discovery to long-term advocacy.
*   **How to use:** Use this to identify friction points. If churn is high, look at the "Convert" or "Retain" stages on the map to figure out where the user is dropping off.

### 4. DVF Assessment (Desirability, Viability, Feasibility)
*   **Function:** The ultimate reality check. It assesses if the product should actually be built.
*   **How to use:** Use this before writing a single line of code. It requires input from UX (Desirability), Business/Finance (Viability), and Engineering (Feasibility).

### 5. Product Roadmap
*   **Function:** A high-level, strategic timeline that sequences the delivery of value (capabilities/themes) over time.
*   **How to use:** Use this to communicate with stakeholders. It tells Sales what they can sell next quarter, and tells Engineering what they should be researching now for future builds.

### 6. PRD (Product Requirements Document)
*   **Function:** The granular blueprint. It details exactly what features must be built to satisfy the current phase of the roadmap.
*   **How to use:** This is the daily working document for engineers. Every Jira ticket should map back to a user story in the PRD.

### 7. Prototype Plan
*   **Function:** Directs the creation of a simulated version of the product to test UX and validate technical assumptions without writing production code.
*   **How to use:** Hand this to a UX/UI designer so they know exactly which screens and flows need to be mocked up in tools like Figma.

### 8. Pitch Deck
*   **Function:** The persuasive narrative that summarizes all previous artifacts to secure funding, resources, or executive approval.
*   **How to use:** Use this to tell a compelling story. It shouldn't be a data dump; it should highlight the problem, the solution, and the business case derived from the DVF.
